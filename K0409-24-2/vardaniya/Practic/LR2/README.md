Научиться собирать Docker-образы, сравнивать их размер и запускать контейнер с ограничениями по CPU и памяти.

## Ход работы

### 1. Подготовка приложения

Было создано простое Flask-приложение с двумя маршрутами:

- `/` — основной ответ приложения
- `/health` — проверка состояния

### 2. Сборка bad-образа

Сначала был создан `Dockerfile.bad`.  
Образ собрался успешно, но получился очень большим.

Команда проверки:

```bash
docker images myapp
```

3. Сборка good-образа

После этого был создан более аккуратный Dockerfile.good.
Сначала возникли ошибки при сборке и запуске:
	•	были проблемы с DNS во время pip install
	•	одна из ранних версий good-образа запускалась с ошибкой ModuleNotFoundError: No module named 'flask'

После исправления Dockerfile.good образ был успешно собран и стал значительно меньше bad-версии.

4. Запуск контейнера с ограничениями

Контейнер app-good был запущен с ограничениями:
	•	память: 128m
	•	CPU: 0.5

Команда запуска:
docker run -d \
  -p 5001:5000 \
  --name app-good \
  --memory="128m" \
  --cpus="0.5" \
  --restart=unless-stopped \
  myapp:good
  Проверка ответа приложения:
  curl localhost:5001

  5. Просмотр слоёв образа

Для good-образа была просмотрена история слоёв:
docker history myapp:good

6. Проверка лимитов контейнера

Для контейнера была просмотрена статистика ресурсов:

docker stats app-good --no-stream

На скриншоте видно, что контейнер работает с ограничением памяти 128MiB.
