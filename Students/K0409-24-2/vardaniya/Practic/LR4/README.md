## Цель работы

Изучить базовую работу Kubernetes: проверку кластера, запуск pod, создание pod через YAML-манифест и проверку самовосстановления контейнера.

## Ход работы

### 1. Подготовка окружения

Сначала в системе отсутствовали `kubectl` и `minikube`, поэтому они были установлены отдельно.  
При первом запуске `minikube` также возникла проблема с нехваткой памяти, поэтому кластер был запущен с меньшим объёмом памяти.

После запуска `minikube` кластер стал доступен для работы.

### 2. Проверка состояния кластера

Сначала была проверена доступность ноды:

```bash
kubectl get nodes
```

После этого были просмотрены системные pod в пространстве имён kube-system:

kubectl get pods -n kube-system

3. Запуск pod nginx

Сначала был создан pod nginx.
При первой попытке с образом nginx:alpine возникла ошибка загрузки образа, поэтому pod был пересоздан с образом nginx.

После этого pod успешно перешёл в состояние Running.

4. Создание pod через YAML

Далее был создан pod my-webserver через YAML-манифест.
В pod использовались два контейнера:
	•	основной контейнер nginx
	•	дополнительный контейнер log-sidecar

После применения манифеста оба pod работали в состоянии Running.

kubectl get pods

5. Проверка самовосстановления

Для проверки self-healing был завершён основной процесс в контейнере nginx внутри pod my-webserver.

Сначала команда kubectl exec my-webserver -c nginx -- kill 1 завершилась ошибкой, потому что kill не был найден напрямую в PATH контейнера. После этого команда была выполнена через shell:

kubectl exec my-webserver -c nginx -- /bin/sh -c 'kill 1'

После этого контейнер был автоматически перезапущен, и значение RESTARTS увеличилось.

kubectl get pod my-webserver

Вывод

В ходе лабораторной работы был успешно запущен локальный Kubernetes-кластер, проверено состояние системных pod, создан обычный pod nginx и pod my-webserver через YAML-манифест.

Также была проверена работа механизма самовосстановления: после завершения основного процесса контейнер внутри pod был автоматически перезапущен, что подтвердилось увеличением счётчика RESTARTS.
