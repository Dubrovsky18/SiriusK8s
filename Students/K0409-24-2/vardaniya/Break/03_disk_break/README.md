## Цель работы

Освободить место на диске без перезагрузки системы и найти процесс, который удерживает удалённый файл.

## Ход работы

Сначала был запущен скрипт `03_disk_break.sh`, который:

- создаёт большой файл-заполнитель `disk_fill.bin`
- создаёт удалённый, но всё ещё открытый файл `orphan_blob`
- запускает фоновый процесс `break_lab_orphan_holder`, который удерживает этот файл

Команда запуска:

```bash
sudo DISK_FILL_GB=2 ORPHAN_MB=256 bash 03_disk_break.sh
```

На скриншоте видно:
	•	путь к файлу /var/lib/linux_break_lab/disk_fill.bin
	•	размер заполнителя 2048 MiB
	•	PID фонового процесса
	•	имя процесса break_lab_orphan_holder

Проверка заполнения диска

После запуска были выполнены команды:


df -h /
du -sh /var/lib/linux_break_lab


На скриншоте видно, что:
	•	файловая система заполнена на 99%
	•	папка /var/lib/linux_break_lab занимает 2,1G

Поиск удалённого, но открытого файла

Для поиска использовалась команда:

sudo lsof | grep deleted | grep orphan

На скриншоте видно, что файл orphan_blob удалён, но всё ещё удерживается процессом.

Освобождение места

Для освобождения места были выполнены действия:
	1.	удалён основной файл-заполнитель
	2.	завершён процесс, удерживающий удалённый файл

После этого была выполнена проверка:

sleep 1
df -h /

На скриншоте видно, что после очистки свободного места стало больше, а использование диска уменьшилось.

Проверка, что удалённый файл больше не удерживается

sudo lsof | grep deleted | grep orphan || echo "deleted orphan file not found"

На скриншоте видно, что удалённый файл больше не удерживается.

Вывод

В ходе работы было показано, как освободить место на диске без перезагрузки системы:
	•	определить, какой каталог занимает много места
	•	найти удалённый, но открытый файл
	•	завершить процесс, удерживающий этот файл
	•	проверить, что место на диске действительно освободилось
