Целью этапа является развертывание базовой микросервисной архитектуры и настройка Nginx в качестве обратного прокси-сервера с обеспечением стабильной маршрутизации трафика и системного мониторинга состояния контейнеров через Health Checks. В процессе конфигурации файла docker-compose.yml и интеграции сервисов выявляется и устраняется ряд инфраструктурных инцидентов. При запуске docker compose up --build -d возникает ошибка checksum mismatch для github.com/lib/pq. Исправляю через go mod tidy в чистом окружении: docker run --rm -v $(pwd):/app -w /app golang:1.21-alpine sh -c "go mod tidy". Сборка проходит. Выполняю задания Lesson 1: смотрю структуру nginx.conf через docker-compose exec nginx cat /etc/nginx/nginx.conf, изучаю upstream backend и параметры limit_req_zone с burst=20, проверяю синтаксис командой docker-compose exec nginx nginx -t, перезагружаю конфиг через nginx -s reload, смотрю логи docker-compose logs -f nginx, тестирую rate limiting — часть запросов получает 503. Поправляю проблему с IPv6: заменяю localhost на 127.0.0.1. Настроен проброс: Nginx:80 → middleware:3000 → backend:8080. Тесты проходят 15 из 16, среднее время ответа /health составляет 0.00095 секунды, заголовки передаются корректно. Задание Lesson 1 выполнено.
