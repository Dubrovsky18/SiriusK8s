Тема: Освобождение дискового пространства (find, xargs, du, lsof)

Что я делала

1. Создание тестовой среды
Так как /data изначально не было, я создала его:

sudo mkdir -p /data
sudo chown $USER:$USER /data
Создала тестовые файлы:

dd if=/dev/zero of=/data/big1.log bs=1M count=150
dd if=/dev/zero of=/data/big2.log bs=1M count=120
touch -t 202001011200 /data/old_small.log
Ошибки и исправления:

случайно перезаписала old_small.log через dd — дата сбросилась на сегодня.

2. Поиск файлов (find)
Команда:

find /data -name "*.log" -type f -size +100M -mtime +7
Ошибки и исправления:

долго ничего не находило, потому что не было одновременно больших и старых файлов;

исправила: создала big_old.log, затем touch -t 202001011200 — сначала dd, потом touch.

3. Подсчёт размера до удаления (du)
Команда:

bash
find /data -name "*.log" -type f -size +100M -mtime +7 -exec du -ch {} + | tail -1
Ошибки и исправления:

сначала пыталась считать после удаления — получила 0;

поняла: сначала считаем, потом удаляем.

4. Удаление через xargs
Команда:

find /data -name "*.log" -type f -size +100M -mtime +7 | xargs rm -f

5. Проверка результата

Тема: Диагностика и восстановление сетевой connectivity (ip, route, DNS, nsswitch)

Что я делала

посмотрела IP-адреса: ip -br a — увидела enp1s0 с IP 192.168.122.120/24;

проверила связь с внешним миром: ping 8.8.8.8 — пакеты проходили (значит, проблема не в самом интерфейсе).

Ошибок не было.

Затем:

-посмотрела таблицу маршрутизации: ip route;

-обнаружила, что нет шлюза по умолчанию (default gateway);

-добавила шлюз: ip route add default via 192.168.122.1.

Настройка DNS (/etc/resolv.conf):

открыла файл cat /etc/resolv.conf — внутри было пусто или невалидные записи;

добавила DNS-сервер Google:
echo "nameserver 8.8.8.8" > /etc/resolv.conf.

Научилась настраивать nsswitch.conf (порядок резолвинга хостов):

в /etc/nsswitch.conf строка с hosts: меняется на hosts: files dns (чтобы сначала смотрел в /etc/hosts, потом в DNS).

Проверка работоспособности
Что делала:

снова ping 8.8.8.8 — всё ок;

curl https://www.avito.ru — сайт отвечает 

Ошибки и исправления:

сначала curl выдавал ошибку соединения — после исправления nsswitch.conf и resolv.conf всё заработало.

Чему я научилась
-проверять маршруты и добавлять шлюз по умолчанию (ip route);

-понимать разницу между проблемами L3 (нет шлюза) и L7 (DNS);

-настраивать DNS вручную через /etc/resolv.conf;

-исправлять порядок резолвинга в /etc/nsswitch.conf;

-проверять доступ через ping и curl -I.
