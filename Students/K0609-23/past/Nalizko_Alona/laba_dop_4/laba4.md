Отчёт по лабораторной работе №4 и практическому заданию 04
В ходе работы я освоила два сценария: захват и анализ сетевого трафика через tcpdump, а также диагностику и исправление проблемных бинарных файлов.

Захват и анализ HTTP-трафика (tcpdump, curl, tmux, TCP handshake). Я научилась захватывать сетевой трафик с помощью tcpdump. Запустила tcpdump в одной панели tmux: sudo tcpdump -i any -w dump.pcap host google.com and port 80. Флаг -w сохраняет дамп в файл, фильтр ловит только HTTP-трафик к google.com. Во второй панели tmux выполнила curl -v http://google.com для генерации запроса.  Нашла три пакета TCP. Затем запустила tcpdump -r dump.pcap -A и нашла HTTP-заголовки. Трудности:не могла разделить tmux на панели — научилась через Ctrl+b %, потом Ctrl+b ←, потом Ctrl+b ".

Диагностика проблемных бинарных файлов (file, chmod, ldd, strace). Скрипт 04_binary_break.sh создал в /opt/break_lab/ несколько сломанных файлов. Для mystery_no_exec добавила права на исполнение через chmod +x, после чего бинарник запустился. Для mystery_bad_interpreter открыла файл в nano и увидела мусорные символы вместо shebang — файл испорчен, восстановить невозможно. not_a_binary оказался обычным текстовым скриптом с #!/bin/bash и echo hello, полностью рабочим. Для проверки зависимостей использовала ldd, для трассировки системных вызовов — strace. Трудности: думала, что проблема в shebang, но файл оказался бинарным
